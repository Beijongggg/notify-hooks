# Release 命名规则

## 版本号

| 变更类型 | 示例 | 说明 |
|---|---|---|
| Bug 修复 | `1.0.0` → `1.0.1` | 小修改、补丁 |
| 新功能 | `1.0.0` → `1.1.0` | 功能增加 |
| 大版本 | `1.x.x` → `2.0.0` | 需作者声明 |
| 测试版 | `v1.1.0-beta` | 后缀 `-beta`，正式版无后缀 |

## 分支

| 分支 | 用途 | 标签示例 |
|---|---|---|
| `master` | 正式版 | `v1.0.0` |
| `beta` | 测试版 | `v1.1.0-beta` |

## 发布流程

1. 开发在 `beta` 分支
2. 测试版打 `vX.Y.Z-beta` 标签
3. 稳定后合并到 `master`，打 `vX.Y.Z` 正式标签

- 测试版: `v1.1.0-beta`
